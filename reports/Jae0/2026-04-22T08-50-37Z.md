# Jae0 — 최종 제출 직전 기획 문서 일괄 커밋

## 1. 주요 변경사항
- `final_submission_overview.md` 및 두 기획안(도둑과 해커 / 자석 실험실 탈출)의 계획 문서 3종 신규 작성, 총 5,878 라인 단일 커밋.
- `thief_hacker/docs/` 하위에 concept·GDD·TDD 3종 문서 추가 (661 + 644 + 651 라인).
- `magnet_lab_escape/docs/` 하위에 concept·GDD·TDD 3종 문서 추가 (656 + 635 + 644 라인).
- `_guidlines-main/` 경로에 concept/GDD/TDD 템플릿 3종 포함 — 가이드라인 원본을 그대로 repo 에 복사한 형태.
- 실제 Unity 프로젝트, `.cs` 소스, 씬·에셋·ProjectSettings 등 **구현 아티팩트는 0 건**.

## 2. 코드 품질 리뷰
리뷰할 코드가 없다. 이 시점에 repo 에 존재하는 산출물은 전부 Markdown 문서이며, Unity 프로젝트의 흔적(`Assets/`, `ProjectSettings/`, `.meta` 파일)이 전혀 보이지 않는다. TDD 내부에 의사코드 블록이 있으나 실제 구현이 아닌 스케치 수준이라 정적 분석 대상에서 제외한다.

다만 문서 구조 자체에서 설계 관점의 위험 신호가 읽힌다. `final_submission_overview.md`는 개인/소규모 개발이면 **자석 실험실 탈출**을 권장하지만, repo에는 도둑과 해커와 자석 실험실 문서가 모두 병렬로 남아 있다. 실제 구현에 들어가기 전 제작 루트를 하나로 고정하고 나머지는 archive로 분리해야 스코프가 흔들리지 않는다.

`magnet_lab_escape` 쪽 문서는 MVP 방, 오브젝트 타입, QA 기준까지 꽤 구체화돼 있다. 남은 문제는 문서의 양이 아니라 Unity 씬·프리팹·입력 액션으로 옮기는 실행 단위가 아직 없다는 점이다. 스캐폴드 없이 문서만 계속 늘리면 좋은 기획이 실제 플레이로 검증되지 못한다.

## 3. 진행도 평가
첫 리포트 기준 전체 진행도 10% 로 판정한다. 컨셉·GDD·TDD 문서 골격은 갖춰졌으나 implementation 축이 완전 공백이고, M2 프로토타입 일정(2026.05~06)까지 남은 시간이 불과 1~2개월이다. 문서만 놓고 보면 30%대로 보일 수 있으나, 코드 착수 지연과 이중 기획이 진행도를 끌어내린다. 커밋도 거대한 단일 import 커밋이라 작업 이력을 분해해서 검토할 수 없다. 방향 수정이 없으면 제출 직전에 "문서는 멋지나 빌드가 없음" 상황이 거의 확실하다.

## 4. 다음 권장사항
- `final_submission_overview.md`의 권고대로 **자석 실험실 탈출**을 실제 제작안으로 고정하고 Unity 프로젝트 루트를 생성하라.
- Unity 프로젝트 스캐폴드를 먼저 올려라 (2022.3 LTS, URP 2D, Input System, 폴더 구조, `.gitignore`). 프로젝트가 없으면 이후 모든 계획이 허상이다.
- 선택한 게임의 **코어 메커닉 1개만** 플레이어블로 만들어라 (브리처 이동+은신 또는 N/S 자력 끌림·밀림).
- MVP 방 1~3의 레벨 설계를 Unity 씬 배치 체크리스트로 전환하라. 문서의 퍼즐 방 표를 실제 오브젝트 배치 작업으로 쪼개야 한다.
- `_guidlines-main` 템플릿 복사본은 archive로 분리하라. 실제 제출 문서와 템플릿 원본이 한 레벨에 섞이면 문서 완성도 판단이 흐려진다.
- 네트워킹은 **미정 상태로 남겨라**. 솔로 기준 Host-Client 구현은 프로토타입 단계에서 가장 늦게 붙이는 게 옳다.

## 5. 문서화 상태
- **design = 8**: 두 후보 모두 컨셉·GDD가 있고, 특히 자석 실험실 탈출은 협동 퍼즐 경험, 코어 루프, MVP 범위가 선명하다. 다만 실제 제작안 하나로 repo 구조가 정리되지는 않았다.
- **technical = 6**: TDD 문서는 클라이언트 구조, 시스템 아키텍처, 네트워크 전제까지 설명하지만 아직 Unity 프로젝트가 없어 검증된 기술 설계로 보긴 어렵다.
- **spec = 6**: `final_submission_magnet_lab_escape_plan.md`와 GDD에 MVP 방, 오브젝트 타입, 일정, QA 기준이 들어 있다. 부족한 것은 문서 자체보다 씬·프리팹·입력 액션으로 옮겨진 작업 체크리스트다.
