# project-alpha — 2주차 리뷰

## 1. 주요 변경사항
- 레벨 데이터 `LevelDataSO` (ScriptableObject)로 정의 — 지난 리뷰 반영.
- 에디터 상에서 레벨을 직접 편집할 수 있는 커스텀 인스펙터 (`LevelDataEditor.cs`).
- 클리어 조건 판정 로직 (`LevelGoal.cs`) — 목표 지점 도달형 + 블록 배치형 2종.
- 이동 tween 추가 (0.12초 DOTween). 체감 개선 확인.
- 스테이지 5개 완성, 난이도 오름차순 배치.

## 2. 코드 품질 리뷰
레벨 데이터 구조는 잘 잡혔다. `LevelDataSO`가 `GridSize`, `Tiles[]`, `Goals[]`을
깔끔하게 분리한 덕에 에디터·런타임 양쪽이 같은 데이터를 본다.

다만 커스텀 에디터 코드(`LevelDataEditor.cs`)가 300줄을 넘었고 대부분이
`EditorGUILayout` 직접 호출이다. 이후 유지보수가 부담될 수 있음 — 필드 추가
시 에디터 코드도 수정해야 하는 구조. 가능하면 `[SerializeField]` + 기본
인스펙터로 가는 게 실용적.

`LevelGoal` 판정 로직이 `Update()`에서 매 프레임 체크하는데, 이벤트 기반
(블록 이동이 발생했을 때만 체크)으로 바꾸면 성능과 디버깅 양쪽에 좋다.

## 3. 진행도 평가
기능 추가는 계획대로. 다만 "레벨 에디터"는 원래 스프린트 목표에 없던
범위(scope creep)다. 재밌는 시도지만, 남은 기간 대비 스테이지 콘텐츠
제작 시간이 줄고 있다.

## 4. 다음 권장사항
- 커스텀 에디터에 더 이상 시간을 투자하지 말 것 — 기본 인스펙터로 충분.
- `LevelGoal` 판정을 이벤트 기반으로 전환.
- 스테이지 10개 확보 목표. 튜토리얼 3개 + 메인 7개가 현실적.
- 사운드 / BGM 에셋 확정 — 후반에 넣으려면 지금 후보 3~5개 리스트업.
