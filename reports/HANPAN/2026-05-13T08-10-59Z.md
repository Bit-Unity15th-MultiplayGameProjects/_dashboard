# HANPAN — 이벤트 수명·QA 복구 스냅샷

## 1. 주요 변경사항
- `BlackAndWhiteGameUI`가 `OnTiebreakerReset` 구독을 `OnDestroy`에서 해제하도록 보강됐다. 기존 라운드·게임 종료 이벤트 해제 목록과 정렬도 맞췄다.
- `BoardLayout3D`는 연장전 리셋 이벤트를 재구독하기 전에 먼저 해제한다. 재바인딩 시 중복 콜백을 줄이는 방향은 타당하다.
- `RoomUI`의 준비 초기화 실패 문구에서 내부 원인 문자열을 UI에 직접 노출하지 않도록 분리했다. 원인은 로그에 남기고 사용자는 행동 가능한 안내를 받는다.
- 흑과 백 사양서와 QA 체크리스트에 9라운드 비동점 종료 시나리오가 복구됐다. W2 항목이 다시 테스트 표에 포함된 점은 명확한 전진이다.

## 2. 코드 품질 리뷰
이번 커밋의 방향은 좋다. 특히 `BoardLayout3D`에서 `OnTiebreakerReset -= ResetBoard` 후 `+=` 하는 방식은 Unity 오브젝트 재바인딩에서 흔히 생기는 중복 이벤트 호출을 직접 차단한다. 왜 중요한가 하면 연장전 리셋은 보드·HUD 초기화와 연결되므로, 콜백이 두 번 호출되면 정상처럼 보이다가 특정 재게임에서만 상태가 어긋나는 디버깅 난도가 높은 버그가 된다.

다만 `BlackAndWhiteGameUI`는 아직 `OnDestroy` 해제에 그친다. 오브젝트가 파괴되지 않고 비활성화되거나, 같은 UI 인스턴스가 새 컨트롤러에 다시 바인딩되는 구조라면 콜백 잔존 가능성이 남는다. 개선하려면 `BindController` 시작 시 기존 컨트롤러를 `UnbindController()`로 정리하고, `OnDisable`과 `OnDestroy`가 같은 해제 함수를 호출하게 만들어야 한다. `RoomUI`는 내부 원인과 사용자 문구를 분리한 점은 적절하지만, “다시 시도”를 안내하면서 실제 재시도 버튼이나 재호출 경로가 없으면 UX 계약이 깨진다.

자료구조/복잡도/할당 관점에서는 이번 diff 범위에서 명확한 지적 없음. 새 LINQ, 반복 컬렉션 복사, hot path 선형 탐색 증가는 보이지 않는다.

## 3. 진행도 평가
구현 경로는 NGO, Relay, Lobby를 쓰되 세션 상태·호스트 마이그레이션·씬 복귀 규칙을 직접 설계한 `hybrid`로 판단한다. 배포 URL 기준 수동 QA가 누적되어 있고 핵심 2인 플레이 루프는 제출 가능한 수준에 도달했다.

진행도는 84로 본다. 이번 커밋은 새 기능보다 직전 평가의 누락을 메우는 성격이 강하지만, 이벤트 수명과 종료 조건 QA는 시연 안정성에 직접 영향을 준다. 남은 위험은 재시도 UX와 재바인딩 수명, QA 증적의 재현성이다.

## 4. 다음 권장사항
- `BlackAndWhiteGameUI`의 이벤트 해제를 `UnbindController()`로 통합하고, 재바인딩·비활성화·파괴 수명에서 모두 같은 경로를 타게 고정해야 한다.
- 준비 초기화 실패 시 “나가기 후 재입장”만 남기지 말고, 준비 초기화 재시도 버튼 또는 자동 재시도 조건을 코드로 연결해야 한다.
- 호스트 마이그레이션 후 방장 변경, 방 목록 재노출, 재입장 순서를 외부 URL 2클라이언트 로그로 남겨야 한다.
- QA 표의 ✅는 빌드 ID, 브라우저 조합, 실행 로그 링크와 묶어야 다음 회귀 때 같은 조건을 재현할 수 있다.
