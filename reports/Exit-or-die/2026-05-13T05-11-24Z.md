# Exit-or-die — Scrollbar 에셋 추가 스냅샷

## 1. 주요 변경사항
- `Assets/Images/Scrollbar.png` 와 Unity import meta 파일이 새로 추가됐다.
- import 설정은 UI Sprite 계열이며 mipmap 비활성, alpha transparency 활성, 단일 slice `Scrollbar_0` 을 포함한다.
- 씬, 프리팹, 스크립트 변경은 없다. 따라서 실제 스크롤 동작이나 RoomList 연결은 아직 확인되지 않는다.
- 기존 네트워크, 입력, 프리팹 중첩, 2인 플레이 검증 backlog 는 이번 커밋에서 해소되지 않았다.

## 2. 코드 품질 리뷰
이번 diff 는 바이너리 UI 에셋과 `.meta` 추가만 포함하므로 C# 구조, 메서드 책임, 예외 처리 품질을 직접 평가할 코드는 없다. 다만 커밋 메시지의 “스크롤바 추가”가 기능 완료를 의미하지는 않는다. 에셋만 추가되고 `Menu.unity` 나 ScrollRect/Scrollbar 프리팹 참조가 연결되지 않으면 런타임에서는 아무 변화가 없다. 다음 변경에서는 실제 UI 연결, anchor, sliced image 타입, 해상도별 표시를 함께 커밋해야 변경 목적이 검증 가능하다.

Unity import 관점에서는 mipmap 비활성, alpha transparency 활성은 UI 이미지에 적절하다. 단, `spriteMode: 2` 에 단일 sub-sprite 를 쓰고 있으므로 실제 Image 컴포넌트가 `Scrollbar_0` 을 참조하는지 반드시 확인해야 한다. 잘못 참조하면 에디터에서는 에셋이 보여도 런타임 UI에는 기본 이미지가 남는다.

자료구조/복잡도 관점에서는 이번 diff 범위에서는 자료구조/복잡도 관련 명확한 지적 없음. 다만 RoomList 를 실제 로비 데이터로 붙일 때 매 갱신마다 모든 row 를 재생성하면 UI GC와 정렬 비용이 커진다. `lobbyId→row` 매핑과 row pool 로 변경분만 갱신하는 구조가 이 프로젝트의 로비 UI 규모에 맞다.

## 3. 진행도 평가
네트워크 구현 경로는 Unity Netcode for GameObjects 기반의 `framework` 로 분류된다. 프레임워크가 연결·동기화 기반을 맡는 만큼, 현재 단계에서는 2인 happy path, UI 진입 흐름, NetworkPrefab 등록, 권한 모델 문서화까지 더 높은 완성도가 요구된다.

이번 커밋은 UI 자산 추가로는 의미가 있지만 수직 슬라이스를 전진시키는 기능 변경은 아니다. 문서와 사양서는 꽤 두껍게 갖춰졌으나, 실제 2인 Game 씬 회귀와 로비 버튼 흐름이 남아 있어 진행도는 54로 본다. 네트워크 설계 부채와 검증 공백이 겹쳐 risk 는 high 가 타당하다.

## 4. 다음 권장사항
- Menu 씬 Back·Join 버튼 클릭 흐름을 실제 LobbyManager 경로에 연결한다.
- RoomList UI를 더미 텍스트가 아닌 실제 로비 데이터 바인딩으로 교체한다.
- 2인 Game 씬에서 이동·공격·사망·관전·시야각 회귀를 검증한다.
- PlayerController 의 이동·공격·체력 책임을 컴포넌트 단위로 재분리한다.
- 플레이어 공격 판정을 서버 권위와 레이어 필터 기준으로 고정한다.
- 몬스터 6종과 계단·문 프리팹을 NetworkPrefabs에 등록하고 스폰 검증한다.
- Scrollbar 스프라이트를 실제 ScrollRect UI에 연결하고 9-slice 변형을 확인한다.

## 5. 문서화 상태
design 은 8점이다. `docs/기획서.md`, `docs/컨셉.md` 에 코어 루프, 씬 흐름, 상호작용 목표가 충분히 적혀 있어 팀원이 방향을 이해할 수 있다. 다만 로비 실패, 카메라 표시 같은 플레이 검증 표는 아직 좁다.

technical 은 7점이다. `docs/기술설계서.md` 와 던전 생성 문서가 NGO, NetworkVariable, 서버 주도 생성 흐름을 설명한다. 하지만 ClientNetworkTransform 의 예측·보정과 이동 권한 정책이 코드 기준으로 닫혀 있지 않아 신규 기여자가 디버깅 기준을 잡기 어렵다.

spec 은 8점이다. 던전·몬스터·상호작용 사양서는 수치, 상태, QA 항목이 있어 테스트 체크리스트로 활용 가능하다. 다만 실제 씬의 로비·스폰·카메라 회귀 체크리스트가 사양 수준까지 내려오지 않았다.
