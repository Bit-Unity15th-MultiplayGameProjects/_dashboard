# GhostMarch_DOTS — 아이템 선택 입력 개선 스냅샷

## 1. 주요 변경사항
- `VerticalSliceItemSelectionBridge`에 Input System 기반 숫자키·넘패드 1/2/3 선택이 추가됐다.
- 선택지 라벨이 단순 효과명에서 현재 레벨과 다음 레벨을 함께 보여주는 형태로 개선됐다.
- 마우스 클릭과 키보드 단축키가 같은 `SelectItem` 경로를 타도록 정리된 점은 좋다.
- handoff 문서에 이번 UI 입력 개선 내용이 기록됐다. 작업 이력 연결은 유지되고 있다.

## 2. 코드 품질 리뷰
이번 변경은 사용자가 실제로 선택지를 고르는 흐름을 전진시킨 점이 분명하다. 특히 `SelectChoiceIndex`가 인덱스 범위를 확인하고 기존 `SelectItem`으로 위임하는 구조는 중복을 줄였고, 숫자 라벨도 수직 슬라이스 데모에서 선택 의미를 훨씬 명확하게 만든다.

다만 위험 신호도 크다. `VerticalSliceItemSelectionBridge`가 여전히 `Time.timeScale`을 직접 저장·복구한다. 이는 Pause UI, 결과 UI, 디버그 정지 같은 다른 정지 소유자와 겹치는 순간 게임을 잘못 재개할 수 있는 구조다. 왜 문제냐면 pause는 전역 상태이고 마지막으로 닫힌 UI가 전체 게임 시간을 소유해서는 안 되기 때문이다. 개선은 `PauseService` 또는 pause token 방식으로 “누가 정지를 요청했는지”를 중앙에서 합산하는 것이다.

자료구조/복잡도 관점에서는 이번 diff가 새 hot-path 할당을 크게 만들지는 않는다. `Update()`는 패널이 열렸을 때만 키 3개를 확인하고 LINQ나 임시 컬렉션을 만들지 않는다. 그러나 아이템 타입이 여전히 `int`와 `switch`로 라벨, 업적 키, 레벨 계산에 흩어져 있다. 현재 비용은 성능보다 계약 불일치 위험이다. `enum` 기반 static table로 효과명·업적 키·최대 레벨·선택 가능 조건을 묶으면 UI와 ECS 이벤트 계약이 같이 안정된다.

## 3. 진행도 평가
네트워킹 코드는 현재 평가 대상이 아니므로 구현 경로는 단일 플레이 DOTS 수직 슬라이스로 본다. 자동 선택 브릿지를 일반 플레이에서 밀어내고 실제 선택 UI를 붙인 흐름은 제출 가능한 데모에 가까워진 전진이다.

다만 smoke 검증이 아직 auto-pick 경로에 의존한다면 실제 입력 UI는 깨져도 CI나 수동 체크에서 늦게 발견된다. 연속 레벨업 후보 재계산과 pause 소유권도 남아 있어 진행도는 60%로 본다. 낮은 레벨의 플레이 루프는 보이지만, 반복 시연 안정성은 아직 확보됐다고 보기 어렵다.

## 4. 다음 권장사항
- 자동 선택 없이 클릭, 숫자키, 넘패드 선택을 Play Mode smoke에 넣어 실제 입력 경로를 검증한다.
- `Time.timeScale` 직접 복구를 제거하고 pause 요청 소유권을 중앙 서비스로 옮긴다.
- 연속 레벨업에서는 선택 이벤트가 반영된 뒤 다음 후보를 다시 계산하도록 흐름을 분리한다.
- 아이템 타입·라벨·업적 키·레벨 제한을 enum table로 묶고 UI 프리팹/해상도 체크리스트를 문서화한다.
- `VerticalSliceAudioBridge`에는 SFX enum 수와 Inspector 배열 길이를 `OnValidate`에서 검증하게 만든다.
- 무한 배경 타일 갱신은 타일 원점이 바뀐 경우로 제한해 불필요한 갱신을 줄인다.

## 5. 문서화 상태
design은 7점이다. `docs/planning/game-design-spec.md`와 migration 문서가 코어 방향과 재구현 의도를 설명하지만, 수치 목표와 플레이 경험 검증 기준은 아직 샘플 사양서 수준으로 촘촘하지 않다.

technical은 8점이다. `docs/architecture/assembly-boundaries.md`, `folder-and-asmdef-layout.md`, serialized baseline 문서가 신규 진입자가 구조를 따라갈 수 있을 만큼 구체적이다. 다만 자동 검증과 실제 CI 연결 근거는 더 필요하다.

spec은 7점이다. `first-vertical-slice.md`, `legacy-runtime-spec.md`, `weapon-equipment-legacy-spec.md`는 기능 기준을 제공한다. 그러나 이번 UI 선택, 배경 타일링, SFX 배열 계약처럼 Inspector와 테스트 체크리스트로 바로 실행할 수 있는 항목은 아직 비어 있다.

## 6. Backlog
- Aseprite·AudioGen 대량 자산은 현재 디렉터리 존재만 확인되고, 출처·라이선스·실제 사용처 문서가 충분히 보이지 않는다.
- Legacy 이동 후 serialized baseline 문서는 있으나, 하드코딩 경로 회귀를 자동으로 잡는 테스트 유지 근거는 부족하다.
- Unity 버전·패키지 업그레이드성 변경은 기능 커밋과 분리하는 원칙을 계속 유지해야 한다.
- 런타임 checker 배경은 문서에도 임시 safety net으로 남아 있으므로 최종 아트 기반 타일링으로 교체해야 한다.
- AudioBridge의 SFX 배열 인덱스 계약과 Inspector 길이 검증은 아직 코드 차원의 방어가 부족하다.
- TextMesh Pro와 ShaderGraph 리소스 대량 변경은 의도와 영향 범위 문서화가 아직 약하다.

## 7. 이전 Backlog 해결
- 실제 아이템 선택 UI로 자동 선택 브릿지를 교체 항목은 해결로 본다. 현재 정상 플레이 경로에는 `VerticalSliceItemSelectionBridge`가 있고, 이번 커밋으로 클릭 외 숫자키 입력까지 같은 선택 경로에 연결됐다.
