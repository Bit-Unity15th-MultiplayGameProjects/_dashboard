# GhostMarch_DOTS — 레벨업 UI와 무한 배경 스냅샷

## 1. 주요 변경사항
- `VerticalSliceInfiniteBackgroundBridge`가 추가되어 카메라 주변에 런타임 checker 타일을 생성하고, 월드 좌표 기준으로 색상 parity를 유지한다.
- 일반 수직 슬라이스 구성에서 `VerticalSliceItemAutoPickBridge`를 `VerticalSliceItemSelectionBridge`로 교체해 레벨업 3지선다 UI를 표시한다.
- 기존 자동 선택 브리지는 비활성화하고, smoke test에서는 명시적으로 다시 켜서 결정적 테스트 흐름을 유지한다.
- handoff와 migration todo 문서에 무한 배경 임시 구현, 레벨업 선택 UI, auto-pick의 테스트 전용 역할이 추가되었다.

## 2. 코드 품질 리뷰
이번 변경은 실제 플레이 감각을 크게 끌어올렸다. 특히 자동 아이템 선택을 UI 선택으로 바꾼 것은 수직 슬라이스의 체감 완성도를 높이는 전진이다. 다만 `VerticalSliceItemSelectionBridge`는 UI 생성, 레벨업 이벤트 큐, DOTS 버퍼 쓰기, 업적 저장, 아이템 라벨 매핑을 한 클래스가 모두 맡는다. 왜 문제냐면 이후 아이템 수가 늘거나 저장 키가 바뀔 때 UI 코드가 게임 규칙 변경의 병목이 된다. 개선은 `ItemType` enum과 아이템 정의 테이블 또는 ScriptableObject로 라벨·최대 레벨·업적 키를 분리하는 것이다.

가장 위험한 부분은 `Time.timeScale` 직접 제어다. 현재는 이미 다른 시스템이 일시정지한 상태에서 레벨업 UI가 열리면 `previousTimeScale`을 1로 복구할 수 있다. 이는 수직 슬라이스에서는 잘 안 보이지만, 메뉴·연출·사망 정지와 결합하면 재현 어려운 상태 버그가 된다. 중앙 pause service 또는 reference-count 방식의 pause token으로 소유권을 분리해야 한다.

자료구조/복잡도 관점에서는 `VerticalSliceInfiniteBackgroundBridge.LateUpdate()`가 매 프레임 모든 타일에 대해 `GetComponent<MeshRenderer>()`, 위치 설정, 머티리얼 재할당을 수행한다. 기본 25개라 당장 치명적이지는 않지만 Update 계열 hot path에 불필요한 컴포넌트 조회가 들어갔다. `MeshRenderer` 배열을 캐시하고, 카메라가 새 타일 원점에 진입했을 때만 재배치하면 코드도 단순해지고 비용도 고정된다.

## 3. 진행도 평가
네트워킹 코드는 이번 범위에 없으므로 해당 없음으로 본다. 현재 평가는 단일 플레이 DOTS 수직 슬라이스 기준이다. 무한 배경의 임시 안전망과 레벨업 선택 UI가 들어가면서 “움직이고 성장하는” 데모 루프는 더 명확해졌다.

진행도는 60으로 본다. 반복 시연 가능한 축은 생겼지만, UI가 런타임 생성과 전역 timeScale에 기대고 있으며 최종 배경 아트도 아직 임시 checker다. 지금은 기능 추가보다 상태 관리와 데이터 분리를 하루 정도 정리해야 이후 확장이 안전하다.

## 4. 다음 권장사항
- 레벨업 선택 UI의 `Time.timeScale` 직접 제어를 중앙 pause 흐름으로 바꿔라.
- 연속 레벨업 큐에서 첫 선택 결과가 다음 후보 계산에 반영되는지 검증하고 수정하라.
- 아이템 타입, 라벨, 업적 키, 최대 레벨을 enum 기반 테이블이나 데이터 자산으로 분리하라.
- 무한 배경은 `Renderer` 캐시와 타일 원점 변경 감지 방식으로 갱신 비용을 낮춰라.
- 런타임 생성 UI를 프리팹화하고 portrait Game View 해상도 체크를 문서에 남겨라.

## 5. 문서화 상태
design 점수는 4다. 이번 문서 갱신은 구현 handoff 중심이며, 레벨업 선택이 플레이어 성장 경험에서 어떤 역할을 하는지에 대한 디자인 의도는 아직 부족하다.

technical 점수는 6이다. `handoff-state.md`와 migration todo가 현재 DOTS 브리지 구성과 임시 구현 상태를 따라가고 있어 인수인계 문서로는 실무 가치가 있다. 다만 pause, UI, 아이템 이벤트 데이터 흐름은 별도 기술 문서로 분리될 필요가 있다.

spec 점수는 4다. “3개 선택지를 띄운다” 수준은 기록됐지만, 후보 추첨 규칙, 최대 레벨, 회복 후보, 업적 키, 연속 레벨업 처리 같은 테스트 가능한 사양은 아직 코드에 묻혀 있다.

## 6. Backlog

- 런타임 checker 배경은 임시 안전망이므로 최종 아트 기반 타일링으로 교체해야 한다.
