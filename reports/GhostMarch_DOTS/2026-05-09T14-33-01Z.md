# GhostMarch_DOTS — DOTS 수직 슬라이스 확장 스냅샷

## 1. 주요 변경사항
- DOTS 수직 슬라이스가 이동·전투·경험치·레벨업을 넘어 웨이브, 아이템, 세이브, 오디오, HUD, GPU 렌더링까지 확장됐다.
- `GhostMarch.Bridge`에서 Legacy 참조를 제거하고 Input/Audio Bridge를 자체 구현으로 바꾼 점은 좋은 방향이다.
- `WaveDifficultyConfig`, `PlayerGrowthConfig`, `VerticalSliceGameConfig`로 하드코딩 수치를 ECS 설정 데이터로 옮긴 것은 유지보수성 측면에서 의미가 크다.
- 컴파일 에러 7건과 asmdef 누락을 후속 커밋에서 고친 것은 필요했지만, 아직 플레이모드 자동 검증은 완료되지 않았다.

## 2. 코드 품질 리뷰
이번 변경의 가장 큰 장점은 Legacy 직접 수정 대신 DOTS 전용 컴포넌트와 Bridge 계층을 분리한 점이다. 특히 `EnemySpawnSystem`에서 적 카운트를 캐시된 `EntityQuery`로 계산하고, 성장·웨이브 수치를 config component로 분리한 결정은 앞으로 밸런싱과 테스트를 쉽게 만든다. 이 방향은 계속 유지해야 한다.

위험 신호도 분명하다. `DamageResultEvent`는 주석상 `DamageTextSpawnSystem`이 읽고 clear하는 계약인데, `SpriteEffectSpawnSystem`도 같은 버퍼를 읽는다. 소비 순서와 clear 책임이 명확하지 않으면 데미지 텍스트와 피격 이펙트가 중복 생성되거나 누락될 수 있다. 해결책은 이벤트 소비를 한 시스템으로 합치거나, 두 소비자를 명시적으로 정렬한 뒤 마지막 소비자만 clear하게 만드는 것이다.

자료구조·복잡도 관점에서는 `VerticalSliceGPURenderBridge`가 핵심 지적 대상이다. `LateUpdate`마다 여러 query에 대해 `ToEntityArray(Allocator.Temp)`를 반복하고, 각 렌더 루프가 `MaxBatch=1023`에서 조용히 절단된다. 현재 비용은 매 프레임 NativeArray 할당과 entity별 `GetComponentData` 반복이다. 재사용 버퍼, chunk 기반 렌더 DTO, 또는 1023개 단위 분할 배치로 바꿔야 이 프로젝트의 생존자류 대량 오브젝트 렌더링에 맞다. 네트워킹 코드는 이번 범위에 없어 해당 없음이다.

## 3. 진행도 평가
이전 대비 전진 폭은 크다. 핵심 루프는 이제 “실험 코드” 수준을 넘어 플레이 가능한 수직 슬라이스에 가까워졌고, 레거시 분리도 실제 코드 의존성 제거 단계까지 왔다. 커밋 메시지도 세션 단위 handoff와 fix 원인이 비교적 잘 남아 있다.

다만 완료율을 높게 잡기는 어렵다. 머티리얼·스프라이트·씬 리소스 할당, 모바일 Game View 검증, 플레이모드 스모크 테스트가 아직 수동 확인 상태다. 컴파일 성공은 중요한 하한선일 뿐 실제 게임 루프 성공을 보장하지 않는다.

## 4. 다음 권장사항
- `DamageResultEvent` 소비 흐름을 먼저 고정한다. 이 문제는 이펙트·데미지 표시·디버그 카운터를 동시에 흔드는 런타임 버그 후보이다.
- `VerticalSliceGPURenderBridge`의 per-frame query allocation과 1023개 절단을 정리한다. 대량 적 스폰이 목표인 구조에서는 지금 고쳐야 비용이 작다.
- 플레이모드 스모크 테스트를 자동화한다. 최소 기준은 시작, 이동, 적 스폰, 명중, 레벨업, 재시작까지 한 번 통과하는 것이다.
- GPU Quad 크기 계산을 단일 기준으로 통일한다. mesh 크기와 matrix scale을 동시에 적용하는 구조는 표시 크기 버그를 만든다.
- 적 접촉 피해의 “초당 피해량”과 “몹 타입 배율”을 분리한다. 단위가 섞이면 밸런스 표와 코드가 서로 다른 게임을 설명하게 된다.

## 5. 문서화 상태
design 점수는 4점이다. 마이그레이션과 구현 전략 문서는 늘었지만, 플레이어 경험 목표, 코어 재미, 적·아이템 선택의 디자인 의도는 아직 사양서 수준으로 설명되지 않는다.

technical 점수는 7점이다. `handoff-state.md`, Legacy disable plan, reference 문서가 실제 협업에 도움이 된다. 다만 자동 검증 절차와 씬 리소스 세팅 기준이 문서만 보고 재현 가능한 수준까지는 아니다.

spec 점수는 6점이다. 업적 코드, 캐릭터 해금, SFX/프레젠테이션 매핑은 개선됐다. 반면 아이템 UI, GPU 머티리얼, SampleScene 적용 절차, 모바일 해상도 QA는 체크리스트로 쓰기엔 아직 빈칸이 많다.

## 6. Backlog
- Legacy 이동 후 하드코딩 경로와 serialized 참조 회귀 테스트 유지: 이동 자체보다 씬·프리팹 참조가 깨지지 않는지 반복 검증하는 장치가 필요하다.
- Unity 버전·패키지 업그레이드성 변경을 별도 커밋으로 격리: 이번 범위에서 해결 근거가 보이지 않아 계속 이월한다.
- TextMesh Pro와 ShaderGraph 리소스 대량 변경의 의도 문서화: 대량 리소스 변경은 원인과 영향 범위를 별도 기록해야 한다.
- Aseprite·AudioGen 대량 자산의 출처와 실제 사용처 명확화: 사용되지 않는 자산이 빌드 크기와 유지보수 비용을 만든다.
- 실제 아이템 선택 UI로 자동 선택 브릿지를 교체: 자동 선택은 임시 검증용이며 최종 UX가 아니다.
- 오디오 SFX 배열 인덱스와 Inspector 길이 검증 추가: 배열 길이 변경 시 런타임 예외가 날 수 있다.

## 7. 이전 Backlog 해결
- 업적 코드와 캐릭터 해금, SFX 인덱스 매핑을 명시적 표로 기록: `deferred-review-findings.md`와 `editor-settings-and-presentation-baseline.md`가 갱신됐고, SaveBridge와 AudioBridge의 매핑도 코드상 분리됐다. 이전에 불명확했던 업적·해금·SFX 연결 관계는 추적 가능한 상태로 개선됐다.
