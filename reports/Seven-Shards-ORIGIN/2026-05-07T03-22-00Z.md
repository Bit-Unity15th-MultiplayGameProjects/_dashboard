# Seven-Shards-ORIGIN — UI 로컬라이제이션 스냅샷

## 1. 주요 변경사항
- `en/ko/ja/zh-Hans/zh-Hant` 로케일 파일이 340여 키 규모로 확장되어 타이틀, 로비, 상점, 다운 상태, 인벤토리, HUD 문자열을 커버하기 시작했다.
- `LocalizedStrings.cs` 를 제거하고 `LocalizedTmpText`, `UiRuntimeResources` 중심으로 런타임 UI 문자열·리소스 경계를 재구성했다.
- `LocalizedTmpBindingAudit`, `UiLocalizationSmokeAudit` 를 추가해 TMP 바인딩과 로컬라이제이션 smoke 검증 기반을 마련했다.
- `GameNetworkManager`, 로비 드라이버, 재접속·씬 로드 UI 메시지가 로컬라이즈되며 네트워크 UX 품질이 한 단계 올라갔다.

## 2. 코드 품질 리뷰
이번 커밋의 방향은 좋다. 하드코딩 UI 문구를 로케일 키로 끌어내고, TMP 바인딩 감사 도구까지 붙인 점은 유지보수성 측면에서 분명한 전진이다. 특히 네트워크 실패, 포트 점유, 씬 로드 타임아웃처럼 플레이어가 실제로 막히는 지점의 문구가 정리된 것은 QA와 사용자 지원에 직접 도움이 된다.

다만 변경 범위가 UI 전반과 네트워크 로비 흐름에 넓게 걸쳐 있어 회귀 위험이 있다. 문자열 키는 컴파일러가 보호하지 못하므로 오타, 누락 키, `{0}` 포맷 인자 불일치가 런타임에서만 터진다. 감사 스크립트를 추가한 것에서 끝내면 부족하다. CI 또는 릴리즈 체크리스트에서 실패 조건으로 고정해야 의미가 있다.

자료구조/복잡도/할당 관점에서는 이번 diff 범위에서 명확한 hot path 지적은 없다. 다만 `Resources/Localization/*.txt` 가 계속 커지고 있으므로, 로딩 시점과 캐시 정책은 문서화해야 한다. 현재 비용 문제라기보다 다음 단계에서 `Resources.Load` 동기 로딩과 key lookup 책임이 흐려질 수 있는 구조적 경고다.

## 3. 진행도 평가
이전 리포트 대비 UI 완성도는 확실히 올라갔다. 다국어 문자열 세트, TMP 바인딩, smoke 감사, 런타임 리소스 분리까지 한 커밋에서 처리한 것은 단순 폴리싱이 아니라 제품화 단계로 가는 기반 작업이다.

진행도는 64%로 본다. 핵심 플레이와 네트워크 UI는 갖춰지고 있으나, Addressable 전환 미정, CJK 실제 폰트 미해결, 로비 prefab fileID 이력, 설계·사양 문서 부족이 남아 있다. 지금은 완성 직전이 아니라 “플레이 가능성을 안정화하는 중반 후반”에 가깝다.

## 4. 다음 권장사항
- CJK 실제 폰트 자산을 연결하고 `TmpEastAsianFontAudit` 로 ja/zh-Hans/zh-Hant 누락 글리프를 실패 처리하라.
- `Resources` 에 남은 로케일·UI 프리팹을 어떤 기준으로 Addressable 로 옮길지 이번 스프린트에서 확정하라.
- `LocalizedTmpBindingAudit`, `UiLocalizationSmokeAudit` 를 수동 도구가 아니라 CI 또는 제출 전 필수 검증으로 묶어라.
- 네트워크 로비, 재접속, 씬 로드 실패 문구 변경은 2인 플레이 회귀 테스트로 실제 화면까지 확인하라.
- `UiRuntimeResources` 조회 실패 시 대체 리소스, 경고 로그, 사용자 표시 정책을 하나로 정리하라.

## 5. 문서화 상태
design 점수는 4점이다. UI/네트워크 구현은 전진했지만, 게임의 코어 경험과 시스템 선택 이유를 설명하는 디자인 문서가 이번 변경에서 보강됐다는 근거는 없다.

technical 점수는 6점이다. `TMP_CJK_FALLBACK_GUIDE`, 리소스 마이그레이션 체크리스트, smoke 기록은 실무적으로 쓸 수 있는 문서다. 다만 네트워크 권한 모델, 런타임 리소스 로딩 흐름, 실패 정책까지 한눈에 보는 아키텍처 문서는 아직 부족하다.

spec 점수는 5점이다. 로컬라이제이션 키 자체는 사양의 원천으로 볼 수 있으나, 화면별 필수 키, 포맷 인자, 지원 언어별 검수 기준이 표준화된 사양서 형태로 정리되지는 않았다.

## 6. Backlog
- `Resources` 기반 UI 프리팹·로케일 대량 로딩은 아직 Addressable 전환 완료 근거가 없다. `UiRuntimeResources` 로 구조는 좋아졌지만 backlog 는 유지한다.
- ja/zh-Hans/zh-Hant 폰트는 감사와 가이드만으로 해결되지 않는다. 실제 폰트 자산 연결과 글리프 검증 통과가 필요하다.
- `NetworkMultiplayerLobbyRoot.prefab` fileID 셔플 이슈는 이번 diff 에서 정리됐다는 증거가 없다. prefab 변경 이력은 계속 추적해야 한다.
- 구현 진척에 비해 디자인·콘텐츠 사양 문서가 얕다. 기능이 늘수록 신규 팀원이 코드에서만 규칙을 역추적하게 될 위험이 커진다.
