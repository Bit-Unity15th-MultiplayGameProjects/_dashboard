# Exit-or-Die_EEN — 초기 스프린트 리뷰

## 1. 주요 변경사항
- Unity 2022.3 LTS 기반 프로젝트 신규 생성 (`ProjectSettings` 정리 완료).
- `Assets/_Project/` 네임스페이스 구조 도입 — 스크립트·씬·프리팹 분리 기준 수립.
- `PlayerController.cs` 초안 작성: `CharacterController` 기반 이동, 점프, 지면 판정.
- 기본 테스트 씬 `Scenes/Sandbox.unity` 추가.
- `.gitignore` / `.gitattributes` (LFS) 설정 정상.

## 2. 코드 품질 리뷰
초반치고 구조가 깨끗하다. `PlayerController`의 입력 처리가 `Update()`에 몰려있긴
하지만, 프로토타입 단계에서는 과한 추상화보다 현재가 낫다. 다만 `Input.GetAxis`
하드코딩은 곧 한계가 올 테니 Input System 패키지 도입 시점을 결정해둘 것.
매직 넘버(이동 속도 `6f`, 점프 힘 `8f`)는 `[SerializeField]` private 필드로
뽑혀있어서 인스펙터 튜닝이 가능하다 — 이 부분은 좋음.

## 3. 진행도 평가
첫 주 목표 "플레이어가 움직이는 테스트 씬" 달성. 팀 3인의 커밋이 균등하게
분포되어 있어 역할 분담이 시작 단계부터 자리잡고 있는 것으로 보인다.

## 4. 다음 권장사항
- Input System 패키지로 마이그레이션 (지금 전환 비용이 가장 싸다).
- `PlayerController`의 중력/낙하 처리는 별도 상태 머신으로 분리 준비.
- 게임 컨셉 문서 1페이지 작성 — 현재 씬만 봐선 게임 방향이 불명확.
