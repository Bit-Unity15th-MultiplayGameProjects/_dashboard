# Rougelike-EndWalker — 서버 네트워킹 확장 스냅샷

## 1. 주요 변경사항
- TDD가 v1.1로 갱신되며 서버 구조, 패킷 포맷, NetworkObject 동기화 흐름이 구체화됐다.
- `NetworkVariableDelta.Payload`가 `List<byte>`에서 `byte[]`로 바뀌어 불필요한 변환 비용이 줄었다.
- `ApplyVariableUpdate`에 owner 검증이 추가되어 클라이언트가 타인 오브젝트를 수정하는 경로가 차단됐다.
- Server의 `.vs`, `bin`, `obj` 산출물이 제거되고 `.gitignore`가 보강됐다. 방향은 옳다.
- README는 Dedicated TCP Server 기준으로 갱신됐지만, 클라이언트 구현은 여전히 진행 중이다.

## 2. 코드 품질 리뷰
네트워크 권한 모델을 Host-Authority에서 Server-Authority로 정리한 점은 좋은 판단이다. 특히 `NetworkObjectManager.ApplyVariableUpdate`에서 `OwnerClientId != senderClientId`를 검사한 변경은 핵심 보안 경계다. 다만 현재 패턴은 실패 시 조용히 `return`한다. 왜 문제인지 명확하다. 네트워크 버그는 재현이 어렵고, 권한 거부·오브젝트 미존재·인덱스 오류가 모두 같은 “아무 일도 안 일어남”으로 보이면 디버깅 시간이 폭증한다. 최소한 debug 로그, 거부 카운터, 테스트 케이스로 실패 이유를 분리해야 한다.

자료구조/복잡도/할당 관점에서는 이번 diff에 명확한 개선이 있다. `Payload`를 `List<byte>`로 들고 다니며 `ms.ToArray().ToList()`와 `delta.Payload.ToArray()`를 반복하던 구조는 네트워크 동기화 hot path에서 불필요한 배열-리스트 복사를 만든다. `byte[]`로 고정한 변경은 현재 비용을 직접 줄이며, 패킷 페이로드라는 데이터 성격에도 더 맞다. 추가 최적화보다 먼저 이 방향을 유지하고, flush 루프에서 대량 델타가 생길 때만 풀링을 검토하면 충분하다.

반면 Unity `.csproj`와 신규 `.sln` 변경은 주의해야 한다. Unity가 재생성하는 IDE 파일의 GUID 변경이 리뷰에 섞이면 실제 기능 변경과 도구 산출물 변경이 분리되지 않는다. 추적할 파일과 무시할 파일을 팀 규칙으로 고정해야 한다.

## 3. 진행도 평가
이전 대비 서버 네트워킹 설계와 일부 구현은 분명히 전진했다. 빌드 산출물 제거도 품질 관리 측면에서 의미 있다. 그러나 게임 완성도 기준으로는 아직 “플레이 가능한 멀티플레이”가 아니라 “서버 기반 구조를 세우는 중”에 가깝다.

위험도는 high다. 네트워킹 코드와 문서상 범위가 빠르게 커졌지만, 통합 검증·플레이모드 테스트·클라이언트 수직 슬라이스가 아직 부족하다. 지금 검증 체계를 잡지 않으면 이후에는 패킷, 씬 전환, 권한, 동기화 문제가 한꺼번에 얽힌다.

## 4. 다음 권장사항
- 서버-클라이언트 2인 접속, 방 생성, 입장, 연결 해제까지 최소 통합 검증을 만든다.
- `NetworkVariable` owner 거부, 오브젝트 없음, 인덱스 오류를 로그와 테스트로 분리한다.
- Unity 생성 `.csproj`/`.sln` 추적 정책을 정하고 불필요한 GUID churn을 제거한다.
- 클라이언트 네트워크 구현은 전체 구조보다 “접속→방→시드 수신” 수직 슬라이스를 먼저 완성한다.
- README의 문서 링크가 실제 경로와 맞는지 확인하고 깨진 링크를 정리한다.

## 5. 문서화 상태
design은 6점이다. 코어 루프와 시스템 목록은 유지되지만, 캐릭터·보스·밸런스 수치가 아직 실무 사양 수준까지 내려오지 못했다.

technical은 7점이다. 이번 TDD 갱신으로 서버 구조, 패킷 포맷, 동기화 흐름은 이전보다 훨씬 읽을 수 있게 됐다. 다만 빌드·실행·테스트 절차가 부족해 신규 인원이 그대로 재현하기에는 빈틈이 있다.

spec은 4점이다. 기능 상태와 일부 데이터 방향은 적혀 있으나, 레벨·노드 확률·캐릭터 수치·QA 체크리스트가 테스트 가능한 표준으로 굳어지지 않았다. `_sample/docs` 수준과 비교하면 아직 사양서보다 개발 메모에 가깝다.
